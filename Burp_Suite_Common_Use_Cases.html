üîó Intercepting & Modifying HTTP Requests

Burp's Proxy tab lets you capture, view, and modify HTTP/S requests in real-time.

Steps:

  1. Configure your browser to use Burp's proxy (usually 127.0.0.1:8080)
  2. Enable Intercept in the Proxy tab
  3. Visit the target application
  4. Modify requests (headers, parameters, cookies) as needed
GET /profile?id=1001 HTTP/1.1
Host: targetsite.com
Cookie: sessionid=abc123

This allows manual testing of parameters and discovering unauthorized access vectors (e.g., IDOR).

üîç Repeating & Automating Requests with Repeater

Use Repeater to resend modified versions of a request and analyze different responses.

Steps:

  1. Send the intercepted request to Repeater
  2. Modify any part (path, headers, body)
  3. Click "Send" and observe the response

This is great for testing for vulnerabilities like XSS, SQLi, and authentication bypass.

POST /login HTTP/1.1
Host: targetsite.com
Content-Type: application/x-www-form-urlencoded

username=admin'--&password=anything

üß™ Testing Input Fields with Intruder

Intruder is Burp‚Äôs automated fuzzing tool, used to test multiple payloads across parameters.

Example Use Cases:

  • Brute-force login credentials
  • Discover hidden parameters or pages
  • Test for injection flaws

Steps:

  1. Send a request to Intruder
  2. Mark the positions to inject payloads
  3. Select the attack type (Sniper, Cluster Bomb, etc.)
  4. Choose a payload list
  5. Launch the attack

Pro Tip: Use Burp BApp Store extensions like Turbo Intruder for more power.

üîê Analyzing Authentication & Session Management

Burp can help evaluate session handling, cookie security, token predictability, and more.

Use Case:

  • Detect if session cookies are missing HttpOnly or Secure flags
  • Test session fixation or token reuse
Set-Cookie: sessionid=xyz123; Path=/; HttpOnly

Try logging in as one user, then switch tokens manually using Repeater to access other accounts.
