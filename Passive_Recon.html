🕶️ Passive Reconnaissance

Stealthy information gathering using open sources — OSINT at its finest

📌 What is Passive Recon?

Passive reconnaissance is the art of gathering intelligence about a target without directly engaging with its systems. This stealthy phase is all about observation without detection. Ethical hackers rely on publicly available information to build a complete profile before taking any active steps.

🎯 Why is Passive Recon Important?

  • ✅ Helps identify potential attack vectors before detection systems are triggered
  • ✅ Reduces risk during the early phases of penetration testing
  • ✅ Builds a complete view of exposed digital footprints
  • ✅ Essential for bug bounty hunters and red teamers alike

🔍 Key Techniques in Passive Recon

1. Google Dorking

Use advanced Google search operators to uncover sensitive data, misconfigurations, or indexed files.

site:example.com filetype:pdf
intitle:"index of" "backup"
site:example.com inurl:login

2. WHOIS Lookups

Discover domain ownership details, registration data, and contact information.

whois example.com

Online tools: DomainTools, Who.is

3. DNS & Subdomain Enumeration

Identify subdomains using certificate transparency logs, DNS tools, and third-party APIs.

crt.sh/?q=%25.example.com
dig NS example.com
amass enum -passive -d example.com

4. OSINT Framework Tools

Use structured toolkits to automate data collection from public sources.

  • theHarvester – Collect emails, subdomains, IPs
  • Maltego – Visual data relationship mapping
  • SpiderFoot – Fully automated OSINT scanning

5. Social Media & Public Data Mining

Collect data from LinkedIn, Twitter, GitHub, and forums for usernames, technologies, and insider info.

  • LinkedIn → Company employees, technologies used
  • GitHub → Code leaks, repo history
  • Twitter → Infrastructure hints, outage reports

🧭 Step-by-Step Passive Recon Workflow

  1. Define the target scope – Domains, IPs, subnets, company names, technologies
  2. Start with Google Dorking – Explore what search engines reveal
  3. Run WHOIS and DNS checks – Collect registrant and infrastructure info
  4. Enumerate subdomains – Use Amass, crt.sh, Sublist3r
  5. Leverage OSINT tools – Automate data collection (SpiderFoot, theHarvester)
  6. Analyze social platforms – Gather human intelligence (HUMINT)
  7. Document all findings – Maintain logs, screenshots, and links

🧪 Real-World Passive Recon Example

Let's say your target is examplecorp.com. Here's how a passive recon might unfold:

# Discover subdomains via certificate transparency
https://crt.sh/?q=%25.examplecorp.com

# Run theHarvester
theHarvester -d examplecorp.com -b bing,linkedin,google

# GitHub username discovery
site:github.com examplecorp

# Use Amass (passive only)
amass enum -passive -d examplecorp.com

💡 Tips & Best Practices

  • 🔐 Always use passive methods first to avoid detection
  • 📎 Document every finding — even ones that seem minor
  • ⚙️ Combine manual searches with automation for scale
  • 🧭 Use a VPN or Tor browser when exploring sensitive sources
  • 🗃️ Create asset maps: domains, IPs, email patterns, GitHub repos
